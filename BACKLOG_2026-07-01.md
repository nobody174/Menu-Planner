## BUGS / FIXES

### B17. App requires relogin / intermittent "Oops!" errors — shipped, still monitoring
- **Found:** Mobile testing, 2026-07-03
- **Fix shipped:** Persistent 30-day sessions (`PERMANENT_SESSION_LIFETIME` + `session.permanent = True`) and `pool_pre_ping`/`pool_recycle` on the Postgres connection (a stale connection after Render idles out was the likely cause of the "Oops!" errors, and possibly of some session flakiness too)
- **Status:** needs a few more days of normal use to confirm it's actually gone. If it recurs, the next thing to check is iOS WebKit's storage-purging behavior for standalone home-screen PWAs (a platform limitation, not something fixable server-side)

### B28. Recipe eu_096 / eu_083 have broken combined-ingredient units — needs manual fix
- **Found:** during the B20 measurement-unit sweep, 2026-07-03
- **Issue:** Two recipes bundle multiple ingredients into a single line with one shared (and nonsensical) unit — not something a mechanical mapping can safely fix:
  - `pack_pasta_noodles.json` recipe `eu_096` — "Béchamel (melk, smør, mel)" has unit `ml/100g/50g`
  - `pack_beef_red_meat.json` recipe `eu_083` — "Smør og melk" has unit `gram/100ml`
- **Fix needed:** Split each into separate ingredient lines with correct individual amounts and units.

### Activity log may be silently dropping entries
- **Found:** while investigating B21 (swap-recipe), 2026-07-03
- **Issue:** Several places call `append_activity_to_db(household, ...)` using `household = current_household()`, which returns an already-detached ORM object from a closed database session — mutating a detached object doesn't persist anything since nothing commits it afterward.
- **Where:** `deployment/flask_app.py`, `current_household()` call sites around lines ~610, 629, 648, 945, 1004, 1018 — needs an audit of which of these need their own open-session-and-commit (like the swap-recipe fix in B21 got).
- **Why it matters:** matches the still-unchecked "Activity log shows correct entries after actions" item in the testing checklist below — some actions may never be logged.
