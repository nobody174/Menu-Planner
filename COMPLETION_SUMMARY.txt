PROJECT OVERVIEW
════════════════════════════════════════════════════════════════════════════════

Pi-Menu is an automated weekly meal planner for a Norwegian household. It scrapes
HelloFresh recipes, generates balanced menus, deduplicates ingredients, and syncs
shopping lists to Microsoft To Do via a Raspberry Pi 2 web server.

Target: 5 people, no oranges/appelsin, varied quick + moderate recipes

2. INGREDIENT DEDUPLICATOR (core/ingredient_deduplicator.py)
   Lines: 230
   Status: Production-ready ✅

   Features:
   • Fuzzy matching (90%+ similarity) using fuzzywuzzy/Levenshtein
   • Loads 180+ pantry staples from pantry_staples.json
   • Filters out household basics: salt, pepper, oil, butter, sugar, water
   • Filters out aromatics: garlic, onion, chili (user buys packaged)
   • Unit normalization: g↔kg, ml↔l, cups↔tbsp, etc.
   • Quantity aggregation: sums ingredients across recipes
   • Auto-categorization: Proteins, Vegetables, Dairy, Pantry, Herbs, Other
   • Allergen preservation through entire pipeline

   Logic Flow:
   • Load recipes for selected 5 dinners
   • For each ingredient:
     - Check fuzzy match against pantry staples → filter if matched
     - Check fuzzy match against existing ingredients → merge if matched
     - Normalize units and aggregate quantities
   • Categorize by type (keyword matching)
   • Output deduplicated shopping list (14-20 unique items)

3. MENU GENERATOR (core/menu_generator.py)
   Lines: 186
   Status: Production-ready ✅

     - Week dates (start/end)
     - 5 dinner entries (day, recipe_id, title, time, difficulty, protein)

TECHNICAL DECISIONS & RATIONALE
════════════════════════════════════════════════════════════════════════════════

1. Local caching over live scraping
   Why: Respects ToS, fast, offline capable, Pi doesn't need internet

2. Fuzzy matching threshold 90%
   Why: Catches variants like "potet"/"potater" but keeps "potet"/"søtpotet" separate

3. Dual-layer orange filter (scraping + generation)
   Why: Defense-in-depth, logging shows why recipes excluded, prevents edge cases

4. 180+ pantry staples
   Why: Reduces shopping list noise, user buys these anyway, configurable

5. Auto-categorization via keywords
   Why: No manual effort, consistent, simple and reliable

6. Unit normalization (g↔kg, ml↔l)
   Why: Intelligently combines quantities, respects different measurement systems

7. 5-dinner menu (Mon-Sat)
   Why: Household preference, Sunday = leftovers/takeout

8. Protein variety algorithm
   Why: Ensures balanced diet, avoids repetition, better for family meals
